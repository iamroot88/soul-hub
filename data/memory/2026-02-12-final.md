## Key Lessons Learned (Critical)

### Rendering Architecture Mistake
I was building scenes in **UE5** but rendering with a **custom Python renderer**. This is fundamentally wrong:
- Python renderer can't produce photorealism
- Wastes UE5's powerful rendering engine
- Results in broken/unrealistic output

### Correct Approach
1. **Build scene in the tool** (UE5 or Blender)
2. **Render with that tool's native engine:**
   - UE5: Movie Render Queue (Windows > Movie Render Queue)
   - Blender: Cycles renderer (headless via CLI)
3. Get quality output because tool's engine does the rendering
